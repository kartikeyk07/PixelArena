FIREBASE FIRESTORE SECURITY RULES
===================================

Copy these rules to your Firebase Console:

1. Go to Firebase Console (https://console.firebase.google.com)
2. Select your project
3. Navigate to Firestore Database → Rules tab
4. Replace all content with the rules below
5. Click "Publish"

---

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper function to check if user is admin
    function isAdmin() {
      return request.auth.token.email == 'kongekartikey007@gmail.com';
    }
    
    // Helper function to check if user is authenticated
    function isAuthenticated() {
      return request.auth != null;
    }
    
    // Users collection
    match /users/{userId} {
      // Users can read/write their own document
      allow read, write: if request.auth.uid == userId;
      // Admins can read all user documents
      allow read: if isAdmin();
    }
    
    // Bookings collection
    match /bookings/{bookingId} {
      // Users can read all bookings (to check availability) but only see their own full details via app logic
      allow read: if isAuthenticated();
      // Users can create bookings (with required fields: userId, gameId, slot, date, zoneId, zoneName)
      allow create: if isAuthenticated() && 
                        request.auth.uid == request.resource.data.userId &&
                        request.resource.data.gameId != null &&
                        request.resource.data.slot != null &&
                        request.resource.data.date != null &&
                        request.resource.data.zoneId != null &&
                        request.resource.data.zoneName != null;
      // Users can update/delete their own bookings
      allow update, delete: if isAuthenticated() && (request.auth.uid == resource.data.userId || isAdmin());
    }
    
    // Zones collection (read-only for users, admin can manage)
    match /zones/{zoneId} {
      // Anyone can read zones
      allow read: if isAuthenticated();
      // Only admins can write (requires: name, location)
      allow write: if isAdmin() &&
                        request.resource.data.name != null &&
                        request.resource.data.location != null;
      
      // Sub-collection: games within zones
      match /games/{gameId} {
        allow read: if isAuthenticated();
        allow write: if isAdmin() &&
                          request.resource.data.name != null &&
                          request.resource.data.pricePerHour != null;
      }
      
      // Sub-collection: menu items within zones
      match /menu/{menuId} {
        allow read: if isAuthenticated();
        allow write: if isAdmin() &&
                          request.resource.data.name != null &&
                          request.resource.data.price != null;
      }
    }
    
    // Games collection (read-only for users, admin can manage)
    match /games/{gameId} {
      allow read: if isAuthenticated();
      // Admins can create/update games (requires: name, zoneId, pricePerHour)
      allow write: if isAdmin() &&
                        request.resource.data.name != null &&
                        request.resource.data.zoneId != null &&
                        request.resource.data.pricePerHour != null;
    }
    
    // Menu collection (read-only for users, admin can manage)
    // Users can view menu items including in booking modals
    match /menu/{menuId} {
      allow read: if isAuthenticated();
      // Admins can create/update menu items (requires: name, price, zoneId)
      allow write: if isAdmin() &&
                        request.resource.data.name != null &&
                        request.resource.data.price != null &&
                        request.resource.data.zoneId != null;
    }
    
    // User stats collection
    match /userStats/{userId} {
      // Users can read/write their own stats
      allow read, write: if request.auth.uid == userId;
      // Admins can read all stats
      allow read: if isAdmin();
    }
    
    // Analytics data (admin only)
    match /analytics/{document=**} {
      allow read, write: if isAdmin();
    }
    
    // Deny all other access
    match /{document=**} {
      allow read, write: if false;
    }
  }
}

WHAT THESE RULES DO:
====================

✓ kongekartikey007@gmail.com (Admin) can:
  - Read and manage all zones, games, and menu items
  - View all user bookings
  - View all user stats
  - Access analytics data
  - Create/edit zones (requires: name, location)
  - Create/edit games (requires: name, zoneId, pricePerHour)
  - Create/edit menu items (requires: name, price, zoneId)

✓ All other authenticated users can:
  - Create bookings (requires: userId, gameId, slot, date, zoneId, zoneName)
  - **Read all bookings** (to check slot availability when booking)
  - View their own bookings (filtered in app UI)
  - Manage their own bookings (update/delete)
  - Read all zones and games
  - View menu items in booking cards (Menu Modal feature)
  - View their own profile and stats

✓ Unauthenticated users:
  - Cannot access any data
  - Will be redirected to login by the app

KEY FEATURES:
=============

📋 Menu Modal for Bookings:
  - Users can view zone menu items when they have an active booking
  - Menu items displayed in a popup modal with name, price, description, category
  - Menu display is read-only for users (admin manages items)

💳 Booking Validation:
  - All bookings must include: userId, gameId, slot, date, zoneId, zoneName
  - This ensures menu modal has correct zone information to display menu items
  - Users can only create bookings for themselves
  - Users can read all bookings to check slot availability (prevents double-booking)

🔍 Slot Availability:
  - When users select a date, the app queries all bookings for that game/date
  - Booked slots are displayed and unavailable for selection
  - This prevents conflicts while maintaining privacy (app filters user-specific data in UI)

🎮 Game & Menu Data:
  - Games require: name, zoneId, pricePerHour
  - Menu items require: name, price, zoneId
  - Both collections maintain data integrity through field validation

IMPORTANT NOTES:
================

1. These rules use email to identify admins
2. Make sure the user "kongekartikey007@gmail.com" is already registered in Firebase Auth
3. The rules are case-sensitive, so the email must match exactly
4. Field validation is enforced:
   - Bookings MUST include: userId, gameId, slot, date, zoneId, zoneName
   - Games MUST include: name, zoneId, pricePerHour
   - Menu items MUST include: name, price, zoneId
   - Zones MUST include: name, location
5. Menu Modal feature requires zoneId and zoneName in booking data
6. **Users can read all bookings** - This is needed for checking slot availability:
   - When selecting a date, the app queries all existing bookings
   - Shows which slots are already booked
   - Prevents double-booking while allowing anyone to check availability
7. After publishing, test the rules with Firebase Console's "Rules Playground" feature
8. For production, consider using Firebase Custom Claims for more robust admin management
